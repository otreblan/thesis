\section{Structures and flow}

FDTD Lucuma is a simulator made in C++26.
It can be used from the command line and from a gui made with Dear ImGui\cite{imgui}.
Internally it is structured using Entity Component System (ECS) with EnTT\cite{entt} and with swappable services using dependency injection.
From which the \texttt{frontend} (cli or gui) consumes one of the \texttt{backends}.
Currently it has 9 backends:

	\centering
	\begin{tabular}{l|c|c|c}
		Precision: & float16 & float32 & float64 \\
		\hline
		CPU sequential & \checkmark & \checkmark & \checkmark \\
		CPU parallel   & \checkmark & \checkmark & \checkmark \\
		GPU (Vulkan)   & \checkmark & \checkmark & \triangle\footnotemark \\
	\end{tabular}
	\caption{FDTD Lucuma's backends}
	\label{tab:nine_backends}
\end{table}
\footnotetext{As of 2026, float64's \texttt{exp()} is not implemented in mesa's amd drivers}

The sequential backend uses plain C++, while the parallel one uses Taskflow\cite{Huang_2022}.
For the GPU backend, FDTD Lucuma implements the FDTD method using Vulkan\cite{vulkanspec} Compute Shaders written in Shader Slang\cite{10.1145/3197517.3201380}.

\begin{figure}[H]
	\centering
	\includesvg[width=1\textwidth,inkscapelatex=false,]{images/graph.svg}
	\caption{FDTD Lucuma's service dependency graph (when the Vulkan backend is selected)}
	\label{fig:fdtd_lucuma_service_graph}
\end{figure}

The flow of the program from the CLI is as follows:
\begin{enumerate}
	\item The program starts.
	\item Basic services are instantiated.
	\item The backend to be instantiated is selected from the command line arguments. (In Figure \ref{fig:fdtd_lucuma_service_graph} you can see all the services that are instantiated by the backend service)
	\item A new ECS entity is created and the simulation state is attached as a component.
	\item The electric sources are attached to new entities.
	\item The simulation iterates for a determined number of timesteps.
	\item The electric sources are deleted.
	\item The simulation state is deleted.
	\item The services are deleted in reverse order of creation.
\end{enumerate}

\section{Compilation targets}
\subsection{Linux (Debian Experimental)}
C++20's modules have not been widely implemented for a substantial amount of time as of 2026.
Thus, a bleeding edge compiler and toolchain is needed.
\inputminted[bgcolor=codeBg, tabsize=4, breaklines=true, linenos=true]{bash}{\subfix{build-debian.sh}}
\subsection{Linux (Arch Linux)}
Arch Linux as a rolling release distribution has no problems providing the latest software.
In addition, the Arch Linux User Repository (AUR) has niche packages.
Even if a software is not packaged, you can just package it yourself.
As of 2026 I am the maintainer of Arch Linux's Shader Slang package\footnote{\url{https://aur.archlinux.org/packages/shader-slang}}.
\inputminted[bgcolor=codeBg, tabsize=4, breaklines=true, linenos=true]{bash}{\subfix{build-arch.sh}}
\subsection{Linux (any other x86 distribution)}
This target primary purpose is to be run on UTEC's HPC cluster Khipu.
But student users do not have the permission to install new packages and installing all FDTD Lúcuma's dependencies
manually would be a time intensive task.
Junest\cite{junest} solves that problem by creating a nested Arch Linux environment that does not need root user privileges to use.
Unlike bare-metal Arch Linux, inside this new environment the version of the GPU driver must be the same version as the host one for the Vulkan acceleration to work.
\inputminted[bgcolor=codeBg, tabsize=4, breaklines=true, linenos=true]{bash}{\subfix{build-junest.sh}}
\subsection{MacOS}
MacOS does not support Vulkan natively.
Instead, it has a layer over Metal named MoltenVK\cite{moltenvk}.
\inputminted[bgcolor=codeBg, tabsize=4, breaklines=true, linenos=true]{bash}{\subfix{build-macos.sh}}
\subsection{Android (Unrooted Termux)}
As of 2025 most Android devices still only support up to Vulkan 1.1\cite{android_vulkan}.
Some extensions required by Ftdd Lucuma are not widely available, which forces it to use fallbacks that are not as performant.
\inputminted[bgcolor=codeBg, tabsize=4, breaklines=true, linenos=true]{bash}{\subfix{build-termux.sh}}
